# 摸鱼热榜技术选型方案

## 1. 项目定位

摸鱼热榜是一个非商用的公开热点链接聚合网站。第一版只聚合公开榜单的标题、来源、排名、热度和原站链接，用户点击后跳转到原网站，不转载正文、不缓存图片视频、不提供站内新闻阅读页。

项目同时提供一个轻量的美股大盘信息区，用于展示标普 500、纳斯达克、道琼斯、纳斯达克 100、VIX 等指数的概览数据。该数据仅供信息展示，不构成投资建议。

## 2. 总体架构

```text
用户浏览器
  -> Nginx
  -> Astro 前端站点
  -> FastAPI 聚合后端
  -> SeeSea 内网数据采集服务
  -> AkShare 美股指数补充
  -> SQLite / 本地 JSON 缓存
```

公网只暴露网站和自有 API：

```text
https://example.com/
https://example.com/api/*
```

不直接暴露：

```text
SeeSea 服务端口
SeeSea internal API
SQLite 文件
服务器管理面板
```

## 3. 前端技术选型

### 推荐方案：Astro + TypeScript

选择 Astro 的原因：

- 网站以内容展示为主，热搜列表和指数卡片适合服务端渲染或静态预渲染。
- Astro 默认输出真实 HTML，比纯 SPA 更利于搜索引擎抓取。
- 可以保留页面的轻量交互，例如主题切换、平台筛选、局部刷新、展开收起。
- 当前项目已有静态 HTML demo，迁移到 Astro 的成本低。

前端栈：

```text
Astro
TypeScript
CSS Modules 或 Tailwind CSS
可选：ECharts 做指数小图
```

第一版建议尽量少用大型 UI 框架。页面设计已经比较定制，直接使用 Astro 组件和 CSS 更容易保持视觉风格。

### 页面规划

```text
/
  简约版首页：核心热搜平台 + 更新时间 + 版本切换提示

/bold
  大胆版首页：Riso/Zine 旧 UI + 热搜矩阵 + 美股大盘

/ui-new/*
  简约版分页面：关于、财经、A 股、美股等 Vercel-like 页面

/trends
  全部热搜，支持平台筛选

/market
  美股大盘概览（如保留独立路由）

/source/[id]
  单个平台热搜列表
```

### SEO 策略

Astro 页面应在服务端渲染时直接包含热搜标题和原站链接：

```text
搜索引擎请求页面
  -> 返回包含热搜列表的 HTML
  -> 浏览器端 JS 再做局部刷新
```

不要让首屏内容完全依赖浏览器端 `fetch()` 后再渲染，否则搜索引擎可能只能看到空壳页面。

必须配置：

- `<title>`
- `<meta name="description">`
- canonical URL
- `sitemap.xml`
- `robots.txt`
- Open Graph 基础字段
- 每条热搜使用真实 `<a href="...">` 链接

## 4. 后端技术选型

### 推荐方案：FastAPI + Pydantic + httpx

选择 FastAPI 的原因：

- SeeSea 和 AkShare 都在 Python/Rust 生态内，FastAPI 接入成本低。
- Pydantic 适合把不同平台的数据整理成统一结构。
- httpx 适合异步请求 SeeSea。
- 后端职责轻，不需要一开始上复杂框架。

后端栈：

```text
FastAPI
Pydantic
httpx
APScheduler 或 FastAPI lifespan 后台任务
SQLite
可选：Redis
```

第一版不建议使用数据库 ORM。SQLite 加少量 SQL 或简单 repository 封装即可。

### 后端职责

FastAPI 不做内容生产，只做聚合、清洗和缓存：

```text
1. 调 SeeSea 获取热搜
2. 调 SeeSea 获取美股行情列表，可选
3. 直接用 AkShare 补美股大盘指数
4. 统一字段格式
5. 缓存最近一次成功结果
6. 失败时返回旧缓存
7. 对前端暴露稳定 API
```

## 5. SeeSea 接入方案

SeeSea 作为内网数据采集层运行，不直接面向用户。

建议调用：

```text
GET /hot/multiple?platforms=weibo,zhihu,baidu,bilibili-hot-search,douyin,v2ex,github-trending-today&latest=true
GET /hot/{platform_id}?latest=true
GET /hot/platforms
GET /api/stock/list/us
```

实际接口路径以 SeeSea 当前运行版本为准。FastAPI 应封装一层 SeeSeaClient，避免前端或业务代码直接依赖 SeeSea 的响应字段。

统一后的热搜结构：

```json
{
  "platform": "weibo",
  "platformName": "微博热搜",
  "title": "热搜标题",
  "url": "https://example.com/original",
  "rank": 1,
  "heat": "123万",
  "source": "weibo",
  "updatedAt": "2026-05-18T10:00:00+08:00"
}
```

内容边界：

- 保留标题、原站链接、平台、排名、热度。
- 不抓正文。
- 不缓存图片、视频、评论、用户信息。
- 不改写为本站原创。
- 前端点击直接跳转原网站。

## 6. 美股大盘数据方案

SeeSea 现有美股能力更偏 `stock_us_spot_em`，适合美股行情列表，不完全覆盖美股大盘指数。

美股大盘建议由 FastAPI 直接调用 AkShare 补充：

```text
SPX / S&P 500
DJI / Dow Jones
IXIC / Nasdaq Composite
NDX / Nasdaq 100
VIX
```

统一后的指数结构：

```json
{
  "symbol": "SPX",
  "name": "S&P 500",
  "price": 5320.12,
  "change": 28.4,
  "changePct": 0.54,
  "url": "https://finance.yahoo.com/quote/%5EGSPC",
  "marketStatus": "closed",
  "updatedAt": "2026-05-18T10:00:00+08:00",
  "disclaimer": "仅供信息展示，不构成投资建议"
}
```

缓存策略：

```text
美股交易时段：1-3 分钟刷新
非交易时段：15-30 分钟刷新
接口失败：返回最近一次成功缓存
```

## 7. API 设计

### 首页聚合

```text
GET /api/home
```

该接口服务默认简约首页 `/`，也服务 `/bold` 大胆版。默认首页可以只展示热榜矩阵；`markets` 字段仍保留给大胆版或财经页面使用。

返回：

```json
{
  "trends": [],
  "markets": [],
  "sources": [],
  "updatedAt": "2026-05-18T10:00:00+08:00"
}
```

### 热搜列表

```text
GET /api/trends
GET /api/trends?platform=weibo,zhihu,baidu
```

### 单平台热搜

```text
GET /api/trends/{platform}
```

### 数据源状态

```text
GET /api/sources
```

返回每个数据源最近更新时间、成功失败状态、缓存年龄。

### 美股大盘

```text
GET /api/market/us
```

### 健康检查

```text
GET /healthz
```

用于 Nginx、部署平台和监控检查。

## 8. 缓存与容错

第一版使用 SQLite 或本地 JSON 缓存即可。

推荐策略：

```text
热搜：5-10 分钟刷新
美股指数：1-3 分钟或 15-30 分钟，按交易时段区分
首页聚合响应：30-60 秒短缓存
失败兜底：返回最近一次成功数据，并标记 stale=true
```

缓存表设计示例：

```text
cache_entries
  key TEXT PRIMARY KEY
  payload TEXT NOT NULL
  updated_at TEXT NOT NULL
  expires_at TEXT NOT NULL
  source_status TEXT NOT NULL
```

错误策略：

- 单个平台失败不影响其他平台。
- SeeSea 整体失败时返回旧缓存。
- 美股数据失败时返回旧缓存并显示延迟状态。
- 前端显示“更新时间”，不要假装实时。

## 9. 部署方案

### 推荐服务器

MVP 推荐：

```text
2 核 8G
40G SSD
2Mbps+
```

该配置适合运行：

```text
Nginx
Astro 静态站点
FastAPI
SeeSea
AkShare
SQLite
Docker Compose
```

如果后续启用 Playwright、Qdrant、向量搜索、本地 LLM，应升级到更高内存或拆分服务。

### Docker Compose 服务

```text
nginx
  对外暴露 80/443

frontend
  Astro 构建产物，可直接由 Nginx 托管

api
  FastAPI，仅 Docker 内网或 127.0.0.1 暴露

seesea
  SeeSea，仅 Docker 内网暴露

redis
  可选，第一版可以不启用
```

推荐第一版先不引入 Redis，降低运维复杂度。

### Nginx 路由

```text
/           -> Astro 静态文件
/api/*      -> FastAPI
/healthz    -> FastAPI
```

SeeSea 不配置公网反向代理。

## 10. 安全与合规边界

网站定位为“公开榜单链接导航”，不是新闻媒体或财经资讯平台。

前端页脚建议：

```text
本站仅聚合公开榜单标题与原站链接，点击后跳转至原网站。内容版权归原平台及原作者所有。
美股大盘数据仅供信息展示，不构成投资建议。
```

备案与上线注意：

- 使用中国大陆服务器需要 ICP 备案。
- 备案描述建议使用“个人工具网站、网址导航、公开链接索引”。
- 不建议使用“新闻、资讯、媒体、日报、舆情、财经资讯”等描述。
- 备案后按要求完成公安联网备案。

内容限制：

- 不转载新闻正文。
- 不提供评论区、投稿、用户生成内容。
- 不做 AI 时政新闻摘要。
- 不做投资建议、荐股、收益预测。
- 不绕过登录或付费墙。

## 11. 第一版实施计划

### 阶段一：后端聚合 API

- 搭建 FastAPI 项目。
- 实现 SeeSeaClient。
- 实现 `/api/trends`。
- 实现本地缓存和失败兜底。
- 接入 5-7 个热搜平台。

### 阶段二：前端迁移

- 用 Astro 迁移现有静态 demo。
- 首页服务端渲染热搜列表。
- 实现平台筛选、刷新、主题切换。
- 增加 SEO 基础标签。

### 阶段三：美股大盘

- FastAPI 接入 AkShare 指数数据。
- 实现 `/api/market/us`。
- 前端在财经页面或 `/bold` 大胆版中展示美股大盘卡片；默认简约首页优先保持热榜扫描效率。
- 添加“数据延迟”和“非投资建议”提示。

### 阶段四：部署上线

- 编写 Docker Compose。
- 配置 Nginx 和 HTTPS。
- 配置日志轮转。
- 配置定时刷新任务。
- 完成备案后绑定正式域名。

## 12. 暂不采用的方案

### 不采用 Next.js

Next.js 功能强，但当前项目不需要复杂的全栈 React、Server Actions 或重交互应用能力。Astro 更轻，迁移现有 HTML 更自然。

### 不采用纯静态 HTML + 浏览器 fetch

纯静态 HTML 开发快，但 SEO 较弱，首屏内容依赖客户端请求，不利于搜索引擎抓取，也不方便做缓存兜底。

### 不直接暴露 SeeSea

SeeSea 包含管理、缓存、内部接口等能力，直接暴露给公网不利于安全和稳定。必须通过 FastAPI 做一层清洗和权限边界。

### 第一版不采用 Redis/PostgreSQL

当前数据量小，SQLite 或本地 JSON 足够。Redis 和 PostgreSQL 可以等访问量或历史归档需求明确后再引入。

## 13. 最终推荐

```text
前端：Astro + TypeScript + CSS
后端：FastAPI + Pydantic + httpx
采集层：SeeSea 内网服务
美股补充：AkShare
缓存：SQLite 起步
部署：Docker Compose + Nginx
服务器：2C8G40G 起步
```

这个方案能兼顾 SEO、开发速度、服务器成本和后续扩展。第一版重点不是做复杂系统，而是稳定地把公开热搜标题和原站链接展示出来，并把 SeeSea 隐藏在后端服务之后。
